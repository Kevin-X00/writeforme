# 自然语言 → 手写文字生成器 方案

## 核心能力

用户用自然语言描述"我需要写什么"，系统自动生成**可直接用于手写的正式文字**，并渲染成手写体排版供用户照抄或打印。

---

## 典型场景

| 用户输入 | 输出 |
|----------|------|
| "帮我写一封请假条，发烧请一天假" | 格式完整的请假条 |
| "帮我写辞职信，想离职休息一段时间" | 正式辞职信 |
| "情人节卡片上写什么给女朋友好" | 一段优美的情书文字 |
| "小孩生病了，帮我写条子给老师请假" | 家长请假条 |

---

## 技术方案

### MVP 架构（最短路径）

**纯前端单页应用** → 零后端，零部署成本

```
用户输入（文字/语音）
      ↓
  LLM API（生成正式文本）
      ↓
  手写体CSS渲染 + 排版
      ↓
  打印 / 截图 / 复制
```

### 技术栈

| 层 | 选型 | 原因 |
|----|------|------|
| 前端 | 纯 HTML/CSS/JS | 单页，GitHub Pages 托管，零成本 |
| LLM | OpenAI API | 跟前面工具复用配置，模型可控 |
| 手写字体 | CSS @font-face 引用免费中文字体 | 行楷/钢笔体，不开销 |
| 布局 | CSS Grid + @media print | 打印友好，直接出纸质版 |
| 语音输入 | Web Speech API（可选增强） | 浏览器原生，无需额外依赖 |

---

## 产品路线

### Phase 1（1 天）— CLI 工具
`npm install -g writeforme`
- 终端输入自然语言 → 自动生成正式文本
- 同时输出 `.html` 手写渲染文件，浏览器打开即可看/打印

### Phase 2（2 天）— Web 单页
- GitHub Pages 部署 free domain
- 输入框 + 场景选择 + 风格选择
- 实时预览手写效果
- 支持打印 / 截图 / 复制

### Phase 3（可选）— 扩展
- Chrome 扩展（在任何网页的文本框旁出现"帮我写"按钮）
- 语音输入集成
